## Why Mailman?

Mailman keeps your HTTP requests under version control, close to your code, and accessible from any terminal session — no GUI, no cloud sync required.

## How it works

<Steps>
  <Step title="Initialize a project">
    Run `mailman init` in your project directory to create a `config.json` and sample request files.
  </Step>
  <Step title="Define your requests">
    Create JSON files in the `requests/` directory describing your HTTP calls — method, URL, headers, and body.
  </Step>
  <Step title="Configure environments">
    Set up `dev`, `prod`, and any other environments in `config.json` with their own variables and base URLs.
  </Step>
  <Step title="Run requests">
    Execute `mailman run requests/get-users.json -e prod` to send the request and see the response instantly.
  </Step>
</Steps>
